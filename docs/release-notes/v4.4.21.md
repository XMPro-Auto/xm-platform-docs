## App Designer

| Change Type | Description |
| --- | --- |
| Fix | _In v4.4.7, I can't configure a data source to use my stored procedure due to an error that the primary key is required._<br>Data sources for stored procedures without a select statement will not require a primary key. |
| Fix | _In v4.4.17, when I change the date range on my_ [_timeseries analysis_](../blocks-toolbox/visualizations/time-series-analysis.md#time-series-analysis-properties)_, the markers I placed lose their data value label._<br>Markers stay visible, along with their value. |
| Fix | _In v4.4.18, when I view a_ [_Data Grid_](../blocks-toolbox/basic/data-grid.md#data-grid-properties) _at runtime, the datetimes stored as UTC are not converted to my local time zone._<br>The Data Grid converts UTC datetime values to user's local time zone. |
| Fix | _When a new level of business role was added, I can no longer see recommendation alerts for my "grandparent" business role._<br>A user belonging to any child business role will have run access to recommendation alerts of the parent business role. |
| Fix | _In v4.4.16, the Chart Block's_ [_Orientation_](../blocks-toolbox/visualizations/chart.md#orientation) _drop-down is empty._<br><br>The Chart's Orientation drop-down contains options for Vertical and Horizontal. |
| Fix | _In v4.4.16, when defining the Color for a chart_ [_series_](../blocks-toolbox/visualizations/chart.md#dataseries)_, I no longer have the option to choose a value from the data source._<br>For a Chart Block with a data source bound, the Series Setting's Color dropdown allows selecting of variables, parameter or columns from the bound data source. |
